## Thesis — Crime prediction with spatial spillovers (CBS neighbourhoods)

This repository contains a notebook-based, end-to-end pipeline that predicts neighbourhood-level registered crime intensity in the Netherlands using CBS neighbourhood (*buurt*) covariates, optionally augmented with exogenous spatial spillover features (Queen contiguity and boundary-length weights).

### Pipeline overview (run order)

- **`1_merge_and_clean_data.ipynb`**: merges raw crime + neighbourhood covariates, constructs join keys, attaches neighbourhood polygons (GeoPackage), exports merged artifacts to `datasets/pre-processing/`, and creates a cleaned tabular dataset for downstream steps.
- **`2_eda_and_preprocessing.ipynb`**: runs untouched-data EDA on the cleaned dataset, then creates and exports the model-ready datasets (base, queen, boundary) and reduced-feature sensitivity variants.
- **`3_modeling_and_analysis.ipynb`**: municipality-grouped train/test split; MVP cross-validation; hyperparameter tuning (RF/XGB); held-out test evaluation;grouped CV robustness comparison; cluster-bootstrap / permutation tests on test errors; optional SHAP; reduced-feature sensitivity; optional spatial choropleth maps; ethics/bias/error analysis.

### Reproducibility & evaluation protocol

- **Target**: `log_crime_count = log1p(crime_count)` (fixed across base / queen / boundary).
- **Held-out test set**: municipality-grouped split (~20% of municipalities), `random_state=42`.
- **Model selection / tuning**: 5-fold GroupKFold by municipality on the training split; RF/XGB tuned with RandomizedSearchCV (`n_iter=30` RF, `n_iter=40` XGB).
- **Grouped CV robustness**: same municipality folds across models/feature sets; reuses tuned configs (no re-tuning inside each fold). Exploratory paired tests on fold-wise RMSE differences.
- **Held-out inference**: cluster-bootstrap CIs and paired cluster permutation tests on test-set RMSE differences (Holm-adjusted).
- **Leakage safety**: median imputation and scaling inside sklearn Pipelines, fit on training data only (per fold during CV; full training split before final test evaluation).
- **Final reporting**: main tables, SHAP, maps, and ethics analysis use held-out test predictions after refitting on the full training split.

### Running the pipeline (recommended)

Run from a clean kernel in order:

1. `1_merge_and_clean_data.ipynb`
2. `2_eda_and_preprocessing.ipynb`
3. `3_modeling_and_analysis.ipynb`

For notebook 3, run through imports → data load → train/test split → MVP CV → RF/XGB tuning → test evaluation → optional grouped CV / SHAP / maps / sensitivity sections. Some tuning assignment cells may be commented in the saved notebook; execute them (or rely on a prior kernel session) before cells that use `rf_best_*` / `xgb_best_*` or export `final_tuned_hyperparameters.csv`.

### Environment setup

Use a local environment (`.venv/` is not committed). **Python 3.12** is recommended.

#### Option A — Conda (recommended for GeoPandas)

```bash
conda env create -f environment.yml
conda activate thesis-spatial-crime
python -m ipykernel install --user --name thesis-spatial-crime --display-name "thesis-spatial-crime"
```

#### Option B — pip

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```
